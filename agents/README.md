# Инструкция быстрого старта агентов

## Что делать прямо сейчас:

### 1. Создаем главного разработчика (самый важный)

```bash
/agent:create lead-developer \
  --description "Ведущий разработчик LaraCash, архитектор и координатор всей разработки. Анализирует требования, декомпозирует задачи, контролирует качество кода и координирует работу других агентов." \
  --context-file "CLAUDE.md" \
  --capabilities "planning,code-review,coordination,architecture"
```

### 2. Проверяем, что агент создан

```bash
/agent:list
```

### 3. Тестируем работу агента

```bash
@lead-developer "Проанализируй текущее состояние проекта LaraCash и предложи 3 улучшения для повышения производительности и удобства пользователей."
```

### 4. Если работает, создаем остальных

```bash
# Фулстэк-разработчик
/agent:create fullstack-developer \
  --description "Фулстэк-разработчик для LaraCash. Реализует полную функциональность от Laravel контроллеров до Bootstrap frontend. Работает с Livewire, MySQL оптимизацией и интеграцией компонентов." \
  --context-file "CLAUDE.md" \
  --capabilities "laravel-development,frontend-backend-integration"

# Тестировщик
/agent:create testing-specialist \
  --description "Специалист по тестированию LaraCash. Пишет Feature/Unit тесты на PestPHP, обеспечивает высокое покрытие кода, тестирует безопасность и производительность." \
  --context-file "CLAUDE.md" \
  --capabilities "pestphp-testing,feature-testing,security-testing"

# Документатор
/agent:create documentation-agent \
  --description "Специалист по документации LaraCash. Обновляет API документацию, пишет пользовательские руководства, поддерживает актуальность технической документации." \
  --context-file "CLAUDE.md" \
  --capabilities "api-documentation,user-guides,technical-documentation"
```

## Пример работы после создания:

```bash
# Запрос разработчику
@lead-developer "Добавить функцию экспорта кешбэков в Excel"

# Ожидаемый результат: агент декомпозирует задачу:
"""
Я проанализировал ваш запрос на экспорт кешбэков.
Декомпозирую на следующие компоненты:

1. Backend API:
   - POST /api/cashback/export эндпоинт
   - Валидация параметров (даты, карты)
   - Генерация Excel файла
   - Оптимизация для больших объемов

2. Frontend:
   - Форма выбора параметров экспорта
   - Индикатор прогресса
   - Обработка ошибок

3. Тестирование:
   - Feature тесты для API
   - Performance тесты для больших данных
   - Тесты безопасности

4. Документация:
   - API документация эндпоинта
   - Руководство пользователя

Распределяю задачи между агентами...
"""

# Автоматическое распределение
@fullstack-developer "Реализуй экспорт кешбэков: POST /api/cashback/export с валидацией дат и выбором карт"
@testing-specialist "Напиши тесты для экспорта: проверка валидации, прав доступа, производительности"
@documentation-agent "Обнови документацию для экспорта кешбэков"
```

## Если что-то не работает:

### Проблема: Команда /agent:create не существует
**Решение:** Используйте альтернативный синтаксис или проверьте версию Claude Code

### Проблема: Агент не видит контекстные файлы
**Решение:** Дайте контекст напрямую:
```bash
@lead-developer "Ты - ведущий разработчик проекта LaraCash (Laravel 11, управление кешбэками). Твоя задача: анализировать требования и координировать разработку. Проектные файлы:
- CLAUDE.md: основная документация
- routes/web.php: роуты приложения
- app/: исходный код

Проанализируй текущее состояние и предложи улучшения."
```

### Проблема: Агент не понимает проектную структуру
**Решение:** Дайте конкретную задачу с указанием файлов:
```bash
@lead-developer "Изучи файл app/Http/Controllers/CashbackController.php и предложи рефакторинг для улучшения производительности и читаемости кода."
```

## Стратегия создания:

1. **Начните с главного разработчика** - он будет координировать остальных
2. **Протестируйте на простой задаче** - убедитесь, что система работает
3. **Постепенно добавляйте других агентов** - по мере надобности
4. **Расширяйте контекст** - если агент не понимает проект, давайте больше информации
